Thoughts on Progressive Enhancement

Progressive enhancement means building a website starting with the most basic, important content and functions first, so the site still works even if a browser does not support every modern feature. After the basic version works, developers can add CSS, JavaScript, animations, and other improvements for users with newer devices and browsers.

Progressive enhancement is important because websites should be usable for as many people as possible. It helps with accessibility, slower internet connections, older browsers, and situations where JavaScript does not load correctly. Instead of making the site depend completely on advanced features, progressive enhancement makes sure the content is still available first.

NASA Mars Rovers Project Revision Reflection

For this revision, I asked a family member to review my first attempt at the NASA Mars Rovers project and give me feedback as a user. The feedback I received was that the website had good information, but some parts could be easier to read and more organized. They also suggested that the images should clearly match each rover and that the navigation should be simple to follow.

Based on that feedback, I revised the project by checking the layout, improving the spacing between sections, and making sure each page had consistent navigation. I also reviewed the image names, alt text, and links to make sure the website worked correctly and was easier to understand.

One thing I think I did well was creating a complete three-page website with a clear topic and consistent design. I included information about the Mars Rover Program, mission history, and frequently asked questions. I also used semantic HTML, an external CSS file, navigation links, images, and a footer disclaimer.

If I had to start over, I would plan the folder structure and image names before writing the code. I would also collect all my images and sources first so that I would not have to go back later and fix broken links or missing files. Planning first would make the project easier to build and revise.

This project helped me learn more about how HTML and CSS work together to create a full website. I also learned that user feedback is important because another person may notice problems that I missed. Revising the project helped me think more about organization, accessibility, and the user experience.
